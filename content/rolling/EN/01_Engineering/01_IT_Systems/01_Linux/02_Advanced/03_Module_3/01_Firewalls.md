# Linux firewalls: Netfilter, iptables, nftables, UFW, firewalld

@section: LPIC-2 map — Module 3 (system and network security)

Typical **LPIC-2 (202 / 206)** coverage plus **RHEL** practice: Netfilter, `nftables`/`iptables`, **firewalld** zones, persistent rules, logging.

On Linux the firewall is not a random daemon: it is the **Netfilter** framework inside the kernel. Tools such as **iptables**, **nftables**, **ufw**, and **firewalld** are front-ends that program Netfilter.

@section: 1. Packet flow: Netfilter hooks

Visualize where a packet can be intercepted:

1. **PREROUTING** — just arrived; **DNAT** / port forwarding happens here.
2. **INPUT** — destined to a local process; filter inbound attacks here.
3. **FORWARD** — host is a router (`ip_forward=1`); filter transit traffic.
4. **OUTPUT** — generated by local processes.
5. **POSTROUTING** — **SNAT** / MASQUERADE before leaving an interface.

@section: 2. iptables: tables, chains, rules

Structure:

* **Tables:** `filter` (default), `nat`, `mangle`, `raw`, `security`.
* **Chains:** map to hooks (`INPUT`, `OUTPUT`, `FORWARD`…).
* **Rules:** match conditions, `-j` target (`ACCEPT`, `DROP`, `REJECT`, `LOG`).

Syntax pattern:

`iptables -t filter -A INPUT [matches] -j TARGET`

**Safe server baseline (conceptual order matters):**

1. Allow loopback: `-A INPUT -i lo -j ACCEPT`
2. Allow established/related: `-A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT`
3. Allow SSH / HTTP / HTTPS explicitly.
4. Set default policy `DROP` on `INPUT` **only after** SSH is allowed (or you lock yourself out).

**Block one IP:** `iptables -I INPUT -s 1.2.3.4 -j DROP` — **first matching rule wins**.

**Persistence:** rules are RAM-only — Debian/Ubuntu: `iptables-persistent`; save with `iptables-save`.

**RHEL:** `iptables-services` legacy; modern systems prefer **nftables** + **firewalld**.

@section: 3. UFW (Ubuntu / Debian)

UFW is a friendly layer over iptables.

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw allow http
sudo ufw allow https
sudo ufw enable
sudo ufw status numbered
sudo ufw delete 3
```

`ufw limit ssh` adds rate limiting (lightweight anti-bruteforce).

@section: 4. nftables

Modern replacement for iptables/ip6tables. Single `nft` CLI, better performance, **sets** for bulk matches.

Example `/etc/nftables.conf`:
```nft
flush ruleset
table inet filter {
    chain input {
        type filter hook input priority 0; policy drop;
        iifname "lo" accept
        ct state established,related accept
        tcp dport { 22, 80, 443 } accept
        ip protocol icmp accept
    }
}
```

Apply: `nft -f /etc/nftables.conf`.

@section: 5. firewalld (RHEL / Fedora)

Zone-based model:

```bash
firewall-cmd --get-default-zone
firewall-cmd --permanent --add-service=https
firewall-cmd --reload
```

Rich rules for port forwarding; integrates with **SELinux** (service definitions).

@section: 6. NAT and forwarding

Enable routing:

```text
net.ipv4.ip_forward = 1
```

**MASQUERADE** for outbound NAT:

```bash
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
```

**DNAT** example:

```bash
iptables -t nat -A PREROUTING -p tcp --dport 8080 -j DNAT --to-destination 10.0.0.5:80
```

@section: 7. Logging

* UFW: `ufw logging on` → `/var/log/ufw.log`
* iptables: add `-j LOG` before final DROP with `--log-prefix`.

@section: Exam essentials

* Default **DROP** on INPUT, always allow **ESTABLISHED,RELATED**.
* Rule **order** matters.
* Know difference **DROP** (silent) vs **REJECT** (ICMP error).

@quiz: You set INPUT policy to DROP and allow HTTP, but outbound browsing from the server fails. Which rule class is most commonly missing?
@option: Allow UDP 443
@correct: Allow ESTABLISHED,RELATED on INPUT so responses to your own requests return
@option: Open port 53 only

@quiz: What is the main benefit of nftables sets compared to thousands of iptables rules?
@option: IPv6 only
@correct: Matching many IPs/ports in one efficient rule
@option: No need for root

@quiz: Which RHEL tool manages firewall zones and `--permanent` rules?
@option: ufw
@correct: firewall-cmd (firewalld)
@option: nft only
